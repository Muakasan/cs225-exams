"Garden Constructor" Feedback

Question Text
The Setup

Consider a garden class that helps manage a garden of flowers (each flower is an instance of a flower class). The garden class stores the flowers in a 4x4 grid (private variable flower ** grid).

The Problem

In the class garden, create a constructor which takes one parameter, an array of flowers (flower *). This constructor must:

  • Allocate memory on the heap for grid to store the flower in a 4x4 grid.
  • Store each flower in one position of the grid (there is no requirement on the position of each flower on the 4x4 grid, except that each flower must be in exactly one spot).

You should assume that the input array will always contain exactly 16 flowers (one for each position). You need only to write the constructor (you do not need to write all of the "big three").

Your code must compile and work without modifying any of the other helper functions defined in the class (however, feel free to call them in your code!).

Compile and Test

A complete Makefile and tester code is provided for you. To compile and test, run:

make
./garden-test

"Garden Destructor" Feedback

Question Text
The Setup

Consider a garden class that helps manage a garden of flowers (each flower is an instance of a flower class). The garden class stores the flowers in a 4x4 grid (private variable flower ** grid).

The Problem

In the class garden, create a destructor. The destructor must free the memory allocated by the constructor provided.

You should assume that grid will always contain be full of exactly 16 flowers (a flower in each position, no need to worry about NULL values). You need only to write the destructor (you do not need to write all of the "big three").

Your code must compile and work without modifying any of the other helper functions defined in the class (however, feel free to call them in your code!).

Compile and Test

A complete Makefile and tester code is provided for you. To compile and test, run:

make
valgrind ./garden-test

If you have freed all memory, the output of valgrind will report a HEAP SUMMARY that shows "0 bytes in 0 blocks" are in use at exit and that "All heap blocks were freed -- no leaks are possible".

==#####== HEAP SUMMARY:
==#####==     in use at exit: 0 bytes in 0 blocks
...
==#####== All heap blocks were freed -- no leaks are possible

"Adding Backpacks" Feedback

Question Text
The Setup

Consider a backpack class that holds up to a fixed number of items. The backpack class has three private variables:

  • int size, the maximum number of items that fits in the backpack.
  • int items, the current number of items in the backpack. There must never be more items in the backpack than the size of the backpack (items <= size).
  • int *item_weights, an array of the weights of the items currently in the backpack. The length of the array will always be at least items length.
The Problem

In the backpack class, overload the + (plus) operator to add two backpack classes.

The result must be a backpack where:

  • the result size is the sum of the size of each individual backpack, and
  • the items (items and item_weights) in the result are the items from both backpacks

Your code must compile and work without modifying any of the other helper functions defined in the class (however, feel free to call them in your code!).

Compile and Test

A complete Makefile and tester code is provided for you. To compile and test, run:

make
./backpack-test

"Subtracting Schedules" Feedback

Question Text
The Setup

Consider a schedule class that stores a set of courses that make up your schedule. The schedule class has two private variables:

  • int count, the current number of courses in the schedule. This variable is the length of the array courses.
  • string *courses, an array of course names.
The Problem

In the schedule class, overload the - (minus) operator to the courses contained in one schedule from the other.

The result must be a schedule that contains only the courses found in the first schedule that are not found on the second schedule.

Your code must compile and work without modifying any of the other helper functions defined in the class (however, feel free to call them in your code!).

Compile and Test

A complete Makefile and tester code is provided for you. To compile and test, run:

make
./schedule-test
