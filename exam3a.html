"Adopting an Animal" Feedback

Question Text

The Setup

Consider an animal shelter that contains two types of animals: dogs and cats. A dog is represented by the Dog class, a cat is represented by a Cat class, and both classes inherit the Animal class.

The animal shelter maintains a singly list of all the animals (nodes with an Animal * pointer that points to the Animal). When you want to adopt an animal from this shelter, you let them know your preference for a dog, cat, or either.

Based on your preference, you adopt the animal closest to the head of the list that matches your preference.

The Problem

In the List class, complete the adoptCat, adoptDog, and adoptAny member function in list.cpp. Each of these functions must:

  1. find the animal closest to the head of the list that meets the function preferences (if a suitable animal exists),
  2. remove that node from the list, and
  3. return a pointer to the Animal (if a suitable animal was found) or NULL if no animal was found

Reference

The List class (list.cpp, list.h) stores a singly linked list of ListNode nodes, using the same structure and variable names as MP3.

The Animal class has a member function getType() that will return "Dog" or "Cat" based on the type of animal.

Compile and Test

A complete Makefile and tester code is provided for you. To compile and test, run:

make
./animal-test

"Meat and Veggies" Feedback

Question Text

The Setup

Consider a eatery where meals are brought out one-by-one (similar to dim sum, except each cart only has a single meal). Each meal is represented by the class Meal and a pointer to each meal, a Meal *, is stored within a node of a singly linked list.

Some meals contains only meat, some contain only vegetables, and some contain both meat and vegetables. Each diner will select the first meal that matches their eating preference.

The Problem

In the List class, complete the chooseMeal member function in list.cpp. The chooseMeal function must:

  1. find the meal closest to the head of the list that meets the given preferences (if a suitable meal exists),
  2. remove the node containing the meal from the list, and
  3. return a pointer to the meal (if a suitable meal was found) or NULL if no meal was found

Reference

The List class (list.cpp, list.h) stores a singly linked list of ListNode nodes, using the same structure and variable names as MP3.

The Meal class has a member functions containsMeat() and containsVegetables() that returned a bool based on if the meal contains meat and/or vegetables.

Compile and Test

A complete Makefile and tester code is provided for you. To compile and test, run:

make
./eatery-test

"Game Night Leaders" Feedback

Question Text

The Setup

Consider a game night for students, where each student is added to a end of a linked list in the order they arrive to the game night. To help organize the games, every 4th student is a leader. For example:

1st     ->  2nd  ->  3rd   ->  4th  -> 5th     -> 6th    ->  ...
Alice   ->  Bob  ->  Carol ->  Don  -> Erin    -> Faythe ->  ...
[Leader]                               [Leader]

The Problem

In the List class, complete the reorganizeList member function in list-p2.cpp. The reorganizeList function must place all non-leaders at the front of the list followed by all leaders, while otherwise maintaining the same order.

Using the example above, the new list must be the following:

2nd  ->  3rd   ->  4th  -> 6th    -> 1st     -> 5th     ->  ...
Bob  ->  Carol ->  Don  -> Faythe -> Alice   -> Erin    ->  ...
                                     [Leader]   [Leader]

Reference

The List class (list-p2.cpp, list.h) stores a singly linked list of ListNode nodes, using the same structure and variable names as MP3.

The ListNode class contains a bool leader that will be set to true when that entry contains a leader.

Compile and Test

A complete Makefile and tester code is provided for you. To compile and test, run:

make
./gameNight-test
